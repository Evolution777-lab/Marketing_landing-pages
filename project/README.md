# Competitor Intelligence Hub

Инструмент конкурентной разведки для команды продуктового маркетинга площадки
электронных торгов и закупок (**B2G / B2B**: 44-ФЗ, 223-ФЗ, 615-ПП,
коммерческие закупки).

Сервис подключается к почте по IMAP, собирает письма конкурентов
(федеральные ЭТП, коммерческие и западные procurement-платформы),
прогоняет их через **GPT-5.5** и показывает аналитику в едином дашборде.

---

## Возможности

- 📥 **Лента** — карточки писем с разбором: УТП, структура оффера, тон, CTA,
  позиционирование относительно нас и рекомендации команде.
- 📊 **Аналитика** — категории, объём за 30 дней, топ отправителей,
  угрозы по неделям, тепловая карта «тип конкурента × УТП», тональность.
- ⚠️ **Угрозы** — письма с высоким/критическим уровнем угрозы, отсортированы
  по важности, с кнопкой «Отметить как просмотренное».
- ⚙️ **Настройки** — IMAP, интервал опроса, ключ OpenAI, проверка подключения.
- Фильтры, поиск, избранное, архив, экспорт в CSV.
- Автосбор по расписанию (APScheduler).

## Быстрый старт (5 минут)

### 1. Установка

```bash
cd project
pip install -r requirements.txt
```

### 2. Настройка пароля приложения Яндекс

> ⚠️ Для IMAP нужен **пароль приложения**, а НЕ основной пароль аккаунта.

1. Зайдите в [Яндекс ID → Безопасность → Пароли приложений](https://id.yandex.ru/security/app-passwords).
2. Создайте пароль для «Почты (IMAP)».
3. В почте включите доступ по протоколу IMAP
   (Настройки → Почтовые программы → «С сервера imap.yandex.ru по протоколу IMAP»).

### 3. Конфигурация

Скопируйте `.env.example` в `.env` и заполните значения **или** задайте всё
позже через UI на вкладке «Настройки».

```bash
cp .env.example .env
```

```env
IMAP_USERNAME=name@yandex.ru
IMAP_PASSWORD=пароль-приложения
OPENAI_API_KEY=sk-...
```

> Без `OPENAI_API_KEY` сервис работает в режиме keyword-фолбэка:
> категоризация по ключевым словам, остальные поля помечаются как
> `requires_api_key`.

### 4. Запуск

```bash
python main.py
```

Откройте **http://localhost:8000**

### 5. Первый сбор

1. Вкладка **Настройки** → проверьте логин/пароль → «Проверить подключение».
2. Нажмите «Собрать письма сейчас» → «Анализировать все неразобранные».
3. Перейдите в **Ленту**.

## Архитектура

```
project/
├── main.py              # FastAPI + APScheduler (роуты, расписание)
├── models.py            # SQLAlchemy модели (SQLite)
├── email_collector.py   # Yandex IMAP: сбор, дедупликация, классификация
├── analyzer.py          # GPT-5.5 пайплайн + keyword-фолбэк
├── gpt-analyst-prompt.txt  # System + user промпт аналитика
├── dashboard.html       # Весь фронтенд (vanilla JS + Chart.js)
├── requirements.txt
├── .env.example
└── README.md
```

**Стек:** Python · FastAPI · SQLite + SQLAlchemy · imap-tools ·
OpenAI SDK (gpt-5.5) · APScheduler · Chart.js.

## API

| Метод | Путь | Назначение |
|-------|------|-----------|
| GET  | `/api/emails` | Список с фильтрами (`category`, `competitor_type`, `threat_level`, `importance_gte`, `date_from`, `date_to`, `search`) |
| GET  | `/api/emails/{id}` | Полные данные письма |
| POST | `/api/emails/{id}/reanalyze` | Повторный анализ GPT-5.5 |
| POST | `/api/emails/{id}/star` | Переключить избранное |
| POST | `/api/emails/{id}/archive` | Переключить архив |
| POST | `/api/emails/{id}/reviewed` | Отметить угрозу просмотренной |
| POST | `/api/collect` | Собрать письма сейчас |
| POST | `/api/analyze/batch` | Анализ всех неразобранных |
| GET  | `/api/analytics` | Агрегаты для графиков |
| GET  | `/api/settings` | Получить настройки |
| POST | `/api/settings` | Сохранить настройки |
| POST | `/api/settings/test-imap` | Проверить IMAP-подключение |
| GET  | `/api/export/csv` | Экспорт писем (с фильтрами) в CSV |

## Анализатор (GPT-5.5)

- model: `gpt-5.5`
- temperature: `0.2`
- max_tokens: `2500`
- response_format: `{"type": "json_object"}`

Промпт системного и пользовательского сообщений находится в
`gpt-analyst-prompt.txt` — его можно редактировать без изменения кода.

---

## Замечания

- База данных SQLite (`competitor_intel.db`) создаётся автоматически при первом
  запуске.
- Настройки, заданные через UI, сохраняются в БД и имеют приоритет над `.env`.
- Письма помечаются прочитанными после сбора; дубли (одинаковые тема+отправитель
  в пределах 24 часов) пропускаются.
